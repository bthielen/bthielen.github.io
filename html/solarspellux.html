SolarSPELL

design challenge

SolarSPELL is non-profit project, bringing a solar-powered learning library to kids in developing countries. This library looks and acts like a normal webpage but is hosted offline, allowing it to be used without wifi. I redesigned the project's entire interface to make it more usable to our target audience.

the original site

home page

  • No search bar present

  • Navigation bar, hero image and content are adhered to 3 different margins, visually off-balancing the site

  • The topic buttons section was not labeled with a heading, making its purpose confusing

  • The buttons had an inconsistent visual reinforcement, as they displayed 4 different icons per subject

content page

  • A search bar was embedded in each page, making it unclear what results the search bar could actually return

  • Folders were marked by a tiny icon and blended with the rest of the content

  • Content was not organized by type (document, audio, etc.), causing all types of files to be scrambled together

  • Content was displayed as links, making it difficult for children's fingers to click

personas

My target user group are children, community members and Peace Corp volunteers residing in developing countries. These countries are Samoa, Puerto Rico, Fiji, Vanuatu and Rwanda with more countries added every year. The SolarSPELL is distributed to elementary schools and high schools. The majority of the target users speak some English and can be expected to have low technical literacy, as few schools have computers.

my ux process

the interface

homepage

Mobile: I placed the search bar on top of the banner image on mobile, integrating it with the visually-appealing background in a way that would allow the search bar's placement to carry over to the content pages without the lower section of the banner.

Desktop: I gave the topics in the navigation bar room to breath by relocating the logo and text to their own row at the top of the screen. This will allow for growth if future topics are added. I added a universal search bar on the upper-right corner. I designed the site's color scheme of blue to fit exactly with the blue of the sky inside the banner and all future banners, allowing the colors to transition seamlessly. I kept the concept of using icon-buttons as quick links to each topic, but I gave each topic its own icon.

drop-down navigation

Mobile: I used the custom icons that previously appeared on the home page as the navigation links on the mobile version. I made the menu slide in from the side and retain some transparency to show the screen behind it to avoid confusing young users by making them think they navigated to another page.

Desktop: I used the transparent gradient effect in both the desktop and mobile drop-downs, upholding visual consistency.

content page

Mobile: This mobile view shows the category accordion closed. I retained the mobile search bar's transparency to show the sun and mountains behind it, but cropped the banner after the search bar. I arranged the controls in rows, distinguishing each row with a different background color.

Desktop: I reimagined the filter system as inviting, clickable buttons instead of cold checkboxes, adding a slight shadow to each button to emphasize its clickable nature. The shadow turns into a subtle highlight and the button changes color when pressed. I retained the banner as a backdrop to the filters, creating smooth consistency between the homepage and content pages. I made each result a clickable box with an icon showing its filetype and gave each file section a header with the same icon.

content page - refining results

Mobile: This mobile view shows the category accordion when clicked. When the user clicks a filter (Natural Habitat) with sub-categories, mobile tabs appear, which are easily switched between the top-level category and Natural Habitat's sub-categories. I retained the concept of filters as buttons from desktop, stacking them in rows according to their width.

Desktop: When the user clicks a filter "Natural Habitat" that contains sub-category, the title "Environment Categories" that was previously centered, turns into a tab that can be toggled back and forth.
